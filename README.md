# Vault Image Press

Obsidianへ保存する前に、画像をブラウザ内だけで圧縮・背景除去するツールです。
`index.html` に「画像圧縮」「背景除去」の2タブがある。

## 画像圧縮タブ

### 使い方

1. `index.html` を Chrome または Edge で開く
2. 画像をドラッグ＆ドロップする
3. `画像を圧縮する` を押す
4. `保存先を選んで一括保存` から、記事と同じフォルダなどを選ぶ

### 推奨設定

- 出力形式: WebP
- 最大横幅: 1600px
- 画質: 82%

### 安全性

- 画像は外部へ送信しません
- APIキーやサーバーは使いません
- 元画像は変更しません

### 注意

- `保存先を選んで一括保存` は Chrome・Edge向けです
- 非対応ブラウザでは `ブラウザから一括ダウンロード` を使ってください
- アニメーションGIF、HEIC、SVGには対応していません

## 背景除去タブ

人物・キャラクターだけを切り抜き、背景を透明にしたPNGを作る。処理エンジンを2つから選べる。

| エンジン | 使う場面 | 精度 | 追加セットアップ |
|---|---|---|---|
| ブラウザ内蔵モデル | 写真・一般的な被写体 | 標準 | 不要（初回のみモデルをCDNから自動DL） |
| ローカルサーバー isnet-anime | アニメ・イラスト調キャラクター | 高い | 別途Pythonサーバーの起動が必要 |

### ブラウザ内蔵モデル

`@imgly/background-removal`（ONNX+WASM、`https://esm.sh` からESMで動的import）を使用。
写真向けの汎用セグメンテーションモデルのため、フラットカラーのイラストでは以下のような誤検出が起きやすい。

- 背景と近い色調の部分（白い服など）が部分的に透過される
- モデルが低信頼度と判断した領域に「穴」ができる

これに対して `refineCutout()` で後処理を行っている。

- 元画像の四隅から背景色を推定（`sampleBackgroundColor`）
- モデルが透明と判定した画素について、推定した背景色との色距離が一定以上（`COLOR_DIST_THRESHOLD = 45`）離れていれば「本来は不透明なはずの誤判定」とみなして復元
- 色が背景と近すぎる部分（服の影など）は原理上この後処理では救えない。この限界に当たった場合はローカルサーバーのisnet-animeエンジンを使うこと

### ローカルサーバー isnet-anime エンジン

アニメ・イラスト特化のセグメンテーションモデル `isnet-anime`（[rembg](https://github.com/danielgatis/rembg)経由）を、ローカルのPythonサーバーで動かして使う。ブラウザ内蔵モデルより明らかに精度が高く、服の内部の穴・袖口の透過・フレーム端で体が切れる構図でのミスなどが起きにくい。

サーバー本体は Vault 外（`D:\StabilityMatrix-win-x64\BgRemover-isnet-anime\`）に設置している。セットアップの詳細はそちらの README を参照。

#### ブラウザ側の操作

1. 「処理エンジン」で「ローカルサーバー isnet-anime」を選択
2. 「ローカルサーバーを起動」ボタンを押す（カスタムURLプロトコル `vipbg://start` 経由でバックグラウンド起動。初回はブラウザの外部プロトコル確認ダイアログが出るので許可する）
3. 状態表示が「起動中です」になったら使用可能
4. 使い終わったら「ローカルサーバーを停止」を押す（押さないとPCを再起動するまでメモリ約1GBを使用し続ける。VRAM/GPUは使用しない＝CPU版onnxruntimeのため）

サーバーが未起動のまま処理しようとすると、エラーメッセージで起動を促す。

## 制作の経緯（メモ）

1. 画像圧縮ツールに「背景除去」タブを追加する要望から開始
2. 当初はComfyUIの `comfyui-lama-remover` 拡張（LaMaベース）を流用する案を検討したが、これは「マスクで指定した領域を周囲になじませて消す」用途（物体除去・インペインティング）であり、「背景を透明にしてキャラだけ残す」目的には合わないと判断
3. ブラウザ完結の `@imgly/background-removal` を採用 → 実写向けモデルのためイラストで精度不足が発覚
4. 後処理（穴埋め、背景色との距離判定）で改善を試みたが、ヒューリスティックゆえに一つの誤判定パターンを直すと別のパターンで新しい誤判定が生まれるモグラ叩きになった
5. 最終的にアニメ特化モデル `isnet-anime`（rembg／Python）によるローカルサーバー方式を追加し、根本的な精度向上を実現
6. 「ブラウザからボタン一つでローカルサーバーを起動・停止したい」という要望に対し、Windowsのカスタムプロトコル（`vipbg://`）とサーバー側`/shutdown`エンドポイントで対応
